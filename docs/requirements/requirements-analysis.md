# 八十！八十！需求分析文档

## 1. 文档概述

### 1.1 文档目的
本文档详细描述"八十！八十！"益智解谜游戏的功能需求、非功能需求和技术约束，为后续架构设计、开发和测试提供依据。

### 1.2 产品名称
**八十！八十！** - 砸墙解谜游戏

### 1.3 目标平台
- PC 网页浏览器
- 微信小游戏
- 抖音小游戏
- 支付宝小游戏
- 快手小游戏（后续）

---

## 2. 产品概述

### 2.1 产品定位
一款物理益智类解谜游戏，玩家通过策略性砸墙，利用物理重心原理使墙体倒塌，在预算限制内完成挑战。

### 2.2 核心玩法循环
```
开始关卡 → 观察墙体结构 → 点击积木砸墙 → 消耗预算 →
物理模拟 → 墙体倒塌 → 闯关成功 → 剩余预算入私房钱 → 下一关
```

### 2.3 目标用户
- 喜欢益智解谜游戏的休闲玩家
- 微信、抖音等小程序平台用户
- 年龄层：15-45 岁

## 4. 非功能需求

### 4.1 性能需求
| 需求 ID | 需求描述 | 目标值 |
|---------|----------|--------|
| NFR-001 | 首屏加载时间 | ≤ 3 秒 |
| NFR-002 | 物理模拟帧率 | ≥ 30 FPS |
| NFR-003 | 砸墙响应延迟 | ≤ 100ms |
| NFR-004 | 单包体大小 | ≤ 10MB（微信限制） |
| NFR-005 | 内存占用 | ≤ 200MB |

### 4.2 兼容性需求
| 平台 | 最低版本 | 目标版本 |
|------|----------|----------|
| PC Chrome | 80+ | 最新 2 版本 |
| 微信小游戏 | - | 最新基础库 |
| iOS Safari | 13+ | 最新 2 版本 |
| Android WebView | 80+ | 最新 2 版本 |

### 4.3 可用性需求
- 支持断点续玩
- 数据自动保存
- 异常退出后恢复进度
- 支持多语言（中文简体优先）

### 4.4 安全需求
- 防止客户端作弊（修改预算、无限道具等）
- 排行榜数据校验
- API 请求频率限制
- 用户数据加密存储

## 5. 技术约束

### 5.1 开发技术栈
- **前端**: Phaser 3 游戏框架
- **后端**: Golang（优先）或 Rust
- **数据库**: PostgreSQL
- **部署**: 云服务器 + CDN

### 5.2 小游戏平台约束
| 平台 | 包体限制 | 性能约束 | 特殊要求 |
|------|----------|----------|----------|
| 微信 | 主包 4MB，总包 20MB | 内存 512MB | 需资质 |
| 抖音 | 主包 4MB，总包 20MB | 内存 512MB | 需版号 |
| 支付宝 | 主包 2MB，总包 10MB | 内存 300MB | 需资质 |

### 5.3 第三方依赖
- Phaser 3 (游戏引擎)
- Matter.js 或 Phaser 内置物理引擎 (物理模拟)
- 平台 SDK（微信、抖音等）
